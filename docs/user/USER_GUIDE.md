# Octopus V2 用户指南

## 产品边界

Octopus 搜索你选择的原始资料，并把结果定位到可核验的页面、段落、表格、单元格、幻灯片、图片或文本行。原始资料只读，内部缓存由应用管理，用户不需要选择 Index 目录。

`2.1.0.dev0` 支持 PDF、文本、Word、Excel、PowerPoint、常见图片以及 ZIP 内这些格式的正文检索。ZIP 成员使用 `archive.zip!/成员路径` 显示，并按需物化到受控的只读缓存。

## 首次使用

1. 启动 Octopus。
2. 填写资料空间名称，选择包含资料的文件夹。
3. 点击“建立资料空间”。
4. 应用立即进入“资料”页，后台同步继续执行。
5. 等待文档状态出现后进入“搜索”。

资料缓存保存在 `%LOCALAPPDATA%\Octopus\workspaces`，任务保存在 `%APPDATA%\Octopus\workspaces`。卸载或重建缓存不会自动删除任务。

## 搜索与页面核对

- 使用文件名、章节名或正文线索搜索；
- 每个文件只显示一次，结果给出命中原因、质量和最佳证据；
- 文件名精确和前缀匹配优先于正文命中；
- PDF 结果可以在右侧查看真实页面、前后翻页和核对片段；
- 文本结果显示定位到的原文；
- 无法可靠定位页码时会明确说明，不会猜测页码；
- 低质量正文不参与正文排名，但文件名仍可搜索。

`Ctrl+F` 可以从任意主页面切回搜索并聚焦搜索框。

## 研究资料包

点击结果或证据检查器中的“加入资料包”，当前证据会进入活动资料包。可以从文献综述、课程报告或自由研究模板开始，也可以让 AI 根据研究目标生成待确认提案。

资料包页可以：

- 修改名称和目标；
- 增删、改名和排序研究分组；
- 编辑证据用途、引用元数据和核验状态；
- 标记是否已人工确认；
- 移除不再需要的证据；
- 选择 GB/T 7714-2015 或 APA 引用样式；
- 导出包含研究正文、BibTeX、任务快照和来源清单的 ZIP。

任务编辑会先写入本机草稿，再使用 revision 保存到服务端。发生并发冲突时，不会直接覆盖另一处修改；界面会保留可恢复的本地草稿。

资料被移动、删除或内容变化后，旧资料包项目不会静默消失。路径可以在内容哈希一致时自动更新；ZIP 成员改名、来源缺失、哈希变化或出现歧义时会标记为“来源待重新确认”。

## 资料页

资料页顶部显示：

- 文档总数；
- 正文可读；
- 部分可读；
- 识别质量低；
- 仅文件信息；
- 处理失败。

“同步”处理新增、修改、移动和删除，变化日志会显示受影响的资料包。单文件右侧的重新处理按钮会强制重新解析该文件，包括需要的 OCR。

长 PDF 处理时会显示当前文件、阶段、页码和总页数。失败文件会保留在列表中，并可重新处理。

## 设置

本地模式不需要 API Key。

启用 AI 资料卡和研究提案前，需要填写服务商、Base URL、模型和 API Key。远程 Base URL 必须使用 HTTPS；只有本机回环地址可以使用 HTTP。密钥保存到 Windows 凭据管理器，不会写入资料空间配置。

“允许发送疑难页面图像”是独立的隐私授权，默认关闭。关闭时仅使用 PDF 文本提取和本地 OCR。不要仅因为开启文本 AI 就默认开启图像授权。

## 备份与恢复

建议备份：

```text
原始资料文件夹
%APPDATA%\Octopus\workspaces\<workspace_id>\tasks
```

以下内容可以重建：

```text
%LOCALAPPDATA%\Octopus\workspaces\<workspace_id>\workspace.sqlite3
%LOCALAPPDATA%\Octopus\workspaces\<workspace_id>\previews
```

V1 `*-Octopus-Index` 保留为迁移和回滚来源。V2 不会继续修改它。

## 隐私

- 本地服务只监听回环地址；
- 原始资料默认不上传；
- 页面图像必须按资料空间明确授权；
- 诊断、导出和打开原文件都由用户主动发起；
- API Key 不写入任务、SQLite 或前端存储。

遇到问题先查看 [故障排查](TROUBLESHOOTING.md)。安装问题见 [Windows 安装说明](WINDOWS_INSTALLATION.md)。
