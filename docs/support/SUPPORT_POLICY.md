# Octopus 支持政策

## 支持范围

v1.0 支持 Windows 11 x64 的每用户本地安装、Raw/Index 分离、桌面/CLI/Local API 核心工作流、文档列出的解析格式和随产品分发的参考 Plugin。网络盘、Windows 10、其他操作系统以及未审阅第三方 Plugin 不在正式支持承诺内。

## 严重度

| 级别 | 定义 | 首要动作 |
| --- | --- | --- |
| P0 | Raw 被修改/丢失、可利用的凭据或内容泄漏、不可恢复的大范围 Index 损坏 | 停止发布/升级，隔离版本，执行紧急回滚 |
| P1 | 核心安装、更新、恢复或本地搜索对支持环境普遍不可用，且无安全绕过 | 阻断发布，准备修复候选或回滚 |
| P2 | 有安全恢复路径的局部故障、性能退化或文档缺口 | 进入最近维护版本评估 |
| P3 | 建议、低影响显示问题或支持边界外请求 | 排入常规计划 |

公开仓库当前提供尽力支持，不承诺付费 SLA 或 7×24 值守。版本发布前 P0/P1 必须为 0；`docs/releases/v0.9-known-issues.json` 是 RC 机器可读登记。

## 报告信息

通过 GitHub Issue 报告时，先提供产品版本、稳定错误码、操作系统版本、复现步骤和是否可恢复。推荐附上经过检查并明确同意分享的本地诊断包；不要粘贴 Raw/Index 路径、文件内容、查询、API Key 或 Bearer token。

## 角色

- Release Operator：执行冻结、构建、签名、校验和、安装验证和 Tag/发布检查；发布时必须由仓库维护者明确指派。
- Support Triage：分级、去敏、复现并决定是否触发 P0/P1 流程；发布时必须由仓库维护者明确指派。
- Recovery Owner：维护回滚工件和公告；可与 Release Operator 为同一人，但发布记录必须写明。

本仓库定义了角色和操作流，但 `0.9.0rc1` 工程候选未记录具体真人姓名，因此不声称已完成真人责任人确认。
