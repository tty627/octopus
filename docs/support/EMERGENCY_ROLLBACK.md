# 紧急回滚手册

## 触发条件

发现 Raw 修改、凭据/内容泄漏、不可恢复 Index 损坏、广泛安装/升级失败或其他 P0/P1 时，停止继续发布并进入回滚。

## 操作顺序

1. 冻结当前 Release/Tag 和升级公告，记录受影响版本、工件 SHA-256、首次发现时间和支持范围。
2. 禁止新用户安装受影响版本；不要删除用户 Raw 或 Index。
3. 若仅配置迁移受影响，先运行 `octopus migrate --rollback RUN_ID`；校验失败时停止，不手工覆盖。
4. 若需二进制回退，使用上一份已验证、哈希匹配且签名有效的安装器。持久化 Schema 超出旧版本支持时，旧版本必须拒绝写入；先执行受支持迁移回滚。
5. 运行 `octopus validate --format json`、本地搜索和 Raw 前后 SHA-256 抽样；保存脱敏诊断包。
6. 发布包含受影响版本、缓解措施、恢复命令和新候选状态的公告。不得把“卸载并删除用户数据”作为默认恢复步骤。

## 关闭条件

- P0/P1 根因已有测试，Raw 修改事件为 0，故障注入恢复通过。
- 替代工件完成签名、校验和、干净环境安装/升级/回退验证。
- Known Issues、CHANGELOG、兼容矩阵和支持公告一致。

自动化能验证命令、契约和工件；停止发布、指定责任人和对外公告需要真人 Release Operator 执行。
