# Octopus v0.7 版本执行文档

## 状态

已规划。

## 目标用户

需要在不修改 Octopus 核心的情况下，将索引组合为打包、时间线等专项工具的开发者和高阶用户。

## 核心问题

第三方扩展能否在明确权限、版本和失败隔离边界内安全复用 Octopus 能力。

## 用户价值

用户可以根据索引完成专项工作流，而不必允许扩展默认读写原文件或内部状态。

## 范围

- Plugin Manifest、能力发现、版本协商、最小权限和进程隔离。
- 只读索引查询、用户授权的导出目录和脱敏运行日志。
- Package 与 Timeline 两个参考 Plugin。

## 非目标

- 公开市场、未审核远程代码自动执行、无限制 Raw 写权限。

## 交付物

- Plugin SDK/CLI、契约文档、权限模型、测试工具和 Package/Timeline 参考实现。

## 指标

- 两个参考 Plugin 仅通过公开契约完成端到端任务。
- 未授权 Raw 或网络访问用例阻断率 100%。

## 验收清单

- [ ] Plugin 兼容性检查在执行前完成。
- [ ] Plugin 崩溃不中断核心更新事务，不泄漏 API Key 或文件内容。
- [ ] Package 只复制用户确认的链接目标，Timeline 只使用允许的 Manifest/索引信号。

## 依赖

v0.6 稳定本地 API、权限与用户确认交互。

## 风险

过早冻结契约会阻碍核心演进；本版本标记为开发者预览，以两个参考 Plugin 验证最小契约。

## 迁移要求

Plugin Manifest 必须声明 API 版本范围；不兼容扩展被禁用并提供升级说明，不执行隐式转换。

## 退出条件

公开契约、权限隔离、版本协商和两个参考 Plugin 全部通过安全/端到端验收。
