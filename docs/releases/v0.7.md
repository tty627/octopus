# Octopus v0.7 版本执行文档

## 状态

工程里程碑已完成。候选版本为 `0.7.0a1`，Plugin API v1 仍标记为开发者预览。

## 目标用户

需要在不修改 Octopus 核心的情况下，将索引组合为打包、时间线等专项工具的开发者和高阶用户。

## 核心问题

第三方扩展能否在明确权限、版本和失败隔离边界内复用 Octopus 能力。

## 用户价值

用户可以根据索引完成专项工作流，而不必允许扩展默认读写原文件、内部状态或环境凭据。

## 范围

- Plugin Manifest、能力发现、版本协商、最小权限和独立子进程。
- 脱敏的只读索引查询/时间线资源、宿主代理的授权导出和确认复制。
- Package 与 Timeline 两个参考 Plugin。

## 非目标

- 公开市场、未审核远程代码自动执行、任意网络访问或 Raw 写权限。
- 把 Python 审计钩子声明为可对抗恶意代码的操作系统安全沙箱。

## 交付物

- Plugin API v1 SDK/CLI 与[契约文档](../plugins/PLUGIN_SDK_V1.md)。
- `index.query`、`index.timeline`、`export.write` 和 `export.copy_confirmed` 权限模型。
- Package/Timeline 参考实现、构建打包规则和端到端/越权测试。

## 指标

- 两个参考 Plugin 仅通过公开请求/响应契约完成端到端任务：2/2。
- 自动化 Raw 越权读取与网络访问阻断：2/2；Raw 修改事件：0。
- v0.7 新增 Plugin 测试：8 项；全仓测试：127 项通过（30+58+39）。
- Ruff 全仓与 strict mypy（38 个源模块）通过。

## 验收清单

- [x] Plugin 兼容性和全部权限授权在创建子进程前完成检查。
- [x] Plugin 崩溃/超时与核心进程隔离，父进程 API Key 不进入子进程环境。
- [x] Package 只复制本次调用明确确认的搜索节点；拒绝计划不留下部分导出。
- [x] Timeline 只使用节点 ID、基名、类型、状态和修改时间等脱敏信号。
- [x] Raw 外部读取、网络、子进程和非响应文件写入有阻断边界与自动化回归。
- [x] 参考 Plugin 随 wheel 打包并由 CLI 发现。

## 依赖

v0.6 稳定 Local API/Engine 边界、可解释搜索结果和节点身份。

## 风险

Python 进程内审计不是完整安全内核，可能存在未覆盖的系统调用或实现绕过。因此本版本只支持内置或用户审阅的本地 Plugin，安全文档不承诺运行任意恶意代码。v0.8 继续加强安装、诊断和回滚边界。

## 迁移要求

Plugin Manifest 必须声明 API 版本范围；不兼容扩展在执行前被禁用，不做隐式转换。入口只能返回声明式操作，旧版直接写导出目录的脚本必须迁移到 `export_text`/`copy_source`。

## 退出条件

Plugin API v1 文档、权限代理、版本协商、Package/Timeline 参考 Plugin、安全回归和打包发现均已通过可复现工程验证。没有进行真人开发者研究，也不声称完成第三方安全审计。
