# Octopus v0.4 版本执行文档

## 状态

内部工程里程碑已于 2026-07-14 关闭。代码、测试、打包链路、跨平台 CI、无签名 Windows
工件和机器化静默安装/卸载/重装验证已通过。产品负责人明确取消真人验收；正式签名、
干净虚拟机、Defender 和 alpha/RC 用户研究未执行，也不声称相关指标达标或已公开发布。
关闭时产品版本为 `0.4.0a1`。

## 目标用户

不熟悉 Python 虚拟环境、pip 和 Node.js 工具链的 Windows 普通用户。

## 核心问题

用户能否在不理解开发环境的情况下，安全安装、初始化仓库并获得第一条有效结果。

## 用户价值

普通用户可在没有 Python、Node.js 和 API Key 的 Windows 11 x64 环境中，通过桌面向导
完成“安装→选择资料/索引→预检→首次索引→搜索→打开结果”，且 Raw 始终只读。

## 范围

- 简体中文 Tkinter 首次运行向导，不扩展本地 HTTP API。
- 同级非嵌套 Index 建议、只读预检、分阶段进度、安全取消和一键重试。
- Markdown、PDF、DOCX、XLSX、PPTX、OCR 图片的确定性内置示例（小于 5 MiB）。
- 向导仓库默认关闭 AI；现有 CLI 和高级配置保持兼容。
- Python 3.12 x64、PyInstaller 6.21 共享 onedir 和 Inno Setup 6.7.1 每用户安装。
- GUI/CLI 稳定版检查；只访问固定 GitHub Latest Release API，不自动下载或安装。
- 本地匿名验收记录与主动导出流程，不上传遥测。

## 非目标

- 自动升级、完整桌面搜索 UI、系统托盘、开机启动、Windows 服务自动安装。
- 本地 HTTP API 稳定化、Plugin SDK、多用户服务器。
- macOS/Linux 安装器、Windows ARM64 工件；Windows 10 仅尽力兼容。

## 交付物

- `Octopus.exe` GUI、`octopus-cli.exe` CLI 与 `octopus.cmd` 命令入口的共享 onedir 工件。
- 单一完整离线安装器、SHA256 校验文件和受保护 CI Authenticode 签名流程。
- 初始化/预检/进度/取消/搜索/结果打开实现及版本化估算系数。
- [Windows 安装说明](../user/WINDOWS_INSTALLATION.md)、
  [故障排查](../user/TROUBLESHOOTING.md)和
  [验收协议](v0.4-acceptance-protocol.md)。

## 依赖

Inno Setup 6.7.1；独立 alpha/RC 受试者与干净 Windows 11 Home/Pro 虚拟机。

## 风险

- OCR/ONNX 使离线包约 300 MiB；使用共享 onedir 避免每次启动解压和双份依赖。
- Windows 文件名大小写不敏感，不能在同一目录同时放置 `Octopus.exe` 与
  `octopus.exe`；因此物理 CLI 为 `octopus-cli.exe`，安装目录提供 `octopus.cmd` 命令。
- 签名和真实用户指标不能由源码测试替代，必须保留为发布阻断项。

## 迁移要求

不提升配置、Manifest、Markdown 或搜索 Schema。安装器和卸载器不删除全局配置、Raw、
Index 或示例资料；重装后通过原 `%APPDATA%\Octopus` 自动识别仓库。v0.3 烟雾测试必须在
正式发布前验证此路径。

未执行的签名、Defender、干净 VM 和真人研究。此关闭只允许进入 v0.5 开发，不等同于
签名公开发布；未执行项可在最终发布工程具备真实条件时补做。
