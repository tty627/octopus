# Octopus v0.4 版本执行文档

## 状态

已规划；不创建集成分支，直到 v0.3 正式退出。

## 目标用户

不熟悉 Python 虚拟环境、pip 和 Node.js 工具链的 Windows 普通用户。

## 核心问题

用户能否在不理解开发环境的情况下，安全安装、初始化仓库并获得第一条有效结果。

## 用户价值

80% 的目标用户可在 10 分钟内完成“安装→选择 Raw/Index→首次索引→打开结果”。

## 范围

- Windows 独立安装包/可执行物、卸载和基础升级检测。
- 初始化向导、预估文件数/时间/AI 调用、进度、取消和普通用户错误信息。
- 无 API Key 的完整基础模式和内置示例仓库。

## 非目标

- 完整桌面搜索 UI、Plugin SDK、多用户服务器和 macOS/Linux 正式安装承诺。

## 交付物

- 可签名的 Windows 发布工件、初始化流程、卸载/故障排查文档和受控用户测试报告。

## 指标

- ≥80% 测试用户在 10 分钟内获得第一条有效结果。
- 首次任务成功率≥95%，Raw 修改事件为 0。

## 验收清单

- [ ] 干净 Windows 虚拟机无 Python/Node 时可安装和运行基础流程。
- [ ] 无 API Key 时索引、更新、校验和本地搜索可用。
- [ ] 取消、空间不足、权限不足和网络 AI 失败有明确恢复指引。

## 依赖

v0.3 性能/正确性门槛，稳定 CLI 进程边界和可重复打包链路。

## 风险

安装包体积受 OCR/ONNX 依赖影响；通过分层功能包、离线基础模式和明确磁盘预估缓解。

## 迁移要求

开发环境用户可保留现有 Index Repository；安装器不覆盖用户全局配置，升级前运行兼容性检查。

## 退出条件

用户激活、首次成功和安全指标达标，干净环境安装/卸载/升级烟雾测试通过。
