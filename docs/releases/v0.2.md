# Octopus v0.2 版本执行文档

## 状态

已完成的内部里程碑；无正式 Git Tag，不追溯伪造发布日期。

## 目标用户

在真实文件仓库上长期更新索引、需要故障可解释和可恢复的内部用户。

## 核心问题

进程中断、派生搜索库失败或 AI 失败时，如何避免暴露半更新索引，并让下一次运行可恢复。

## 用户价值

用户可信任更新失败不会损坏原文件或留下无法解释的索引状态。

## 范围

- Manifest-last 事务、回滚意图、恢复与派生缓存重建。
- 不可覆盖的每次运行报告和脱敏 AI 用量。
- dry-run、仓库校验、Provider 错误分类、重试和调用预算。

## 非目标

- 安装器、GUI、检索评测集、大仓库性能优化和 Plugin 契约。

## 交付物

- 事务、报告和校验实现及测试。
- [ADR 0002](../adr/0002-recoverable-index-transactions.md)。

## 指标

- 已定义故障注入点恢复/回滚率 100%。
- Raw 修改事件为 0，运行报告不包含 API Key 或文档摘录。

## 验收清单

- [x] Manifest 在普通索引文件之后提交。
- [x] Manifest 前失败可回滚，Manifest 后失败可重建派生状态。
- [x] 每次尝试生成独立 RunReport。
- [x] dry-run 不写 Raw 或 Index。

## 依赖

v0.1 的 Manifest、Markdown 源数据和可重建 SQLite 缓存边界。

## 风险

事务记录全量重写导致规模性能下降，在 v0.3 作为发布阻断项处理。

## 迁移要求

无对外迁移承诺；事务记录与运行报告均存储在 Index `.octopus/` 下。

## 退出条件

事务、回滚、Manifest 后恢复、报告、dry-run 和校验已由自动化用例覆盖，已退出。
