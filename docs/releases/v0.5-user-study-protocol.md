# Octopus v0.5 定位时间用户研究协议

## 目标与样本

- 至少 20 名已经拥有可用 Octopus 索引的知识工作者。
- 比较 Windows 文件资源管理器与 Octopus 本地确定性搜索的目标文件定位时间。
- 使用固定版本的合成任务集；正式报告记录产品、算法、数据集、环境和日期范围。

## 设计

- 每位参与者先完成不计分的练习任务，再完成 12 个正式任务。
- 任务顺序按匿名参与者 ID 确定性随机；两种条件各 6 个并交替分配。
- 同一参与者不在两种条件中重复同一个目标，避免记忆效应；跨参与者反平衡条件。
- 计时从显示任务并确认开始，到参与者打开目标文件或确认失败结束。

## 本地记录

使用 `octopus evaluate study` 生成 JSONL。记录仅包含匿名 session/participant ID、产品与
数据集版本、任务 ID、条件、耗时、成功状态和错误码；不得记录查询、文件路径、URI、
文件名或内容。`octopus evaluate summarize` 拒绝混合版本、重复记录和包含禁用字段的数据。

## 通过标准

- 两种条件分别报告尝试数、成功数、成功率和成功任务耗时 P50/P95。
- 主要指标为 `(Explorer P50 - Octopus P50) / Explorer P50`，必须不低于 50%。
- 少于 20 名参与者、缺少任一条件或数据版本混用时不得用于关闭 v0.5。
- 原始匿名记录、汇总 JSON 和研究环境说明随候选版本报告保存，不上传文件内容。
