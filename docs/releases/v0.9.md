# Octopus v0.9 版本执行文档

## 状态

已规划；Release Candidate 阶段。

## 目标用户

准备升级到 v1.0 的 Beta 用户、发布工程和支持人员。

## 核心问题

当前产品是否已无阻断 v1.0 稳定性、安全、升级和支持承诺的问题。

## 用户价值

用户获得功能冻结、可预期升级和只聚焦重要问题修复的候选正式版。

## 范围

- 修复 P0/P1、发布门槛、升级阻断、安全和数据一致性问题。
- 冻结用户工作流、持久化 Schema、公开 API 和 Plugin 契约。
- 完成安装、升级、备份、恢复、已知问题和支持文档。

## 非目标

- 新的解析格式、新搜索模式、新 UI 主工作流或新 Plugin 能力，除非它们是修复发布阻断的唯一方案。

## 交付物

- RC 安装包、候选发布报告、完整兼容矩阵、支持手册和已签署发布清单。

## 指标

- 已知 P0/P1 为 0；所有 v1.0 指标和性能门槛通过。
- 全量自动化和手工发布清单通过率 100%。

## 验收清单

- [ ] 至少两个 RC 候选的安装/升级/回滚练习通过。
- [ ] 包、Tag、CHANGELOG、文档、Schema 和签名工件版本一致。
- [ ] 发布支持和紧急回滚责任人/操作流已确认。

## 依赖

v0.8 公开 Beta 指标和真实升级数据。

## 风险

功能蔓延会让 RC 无法稳定；通过变更冻结和“仅发布阻断”准入规则缓解。

## 迁移要求

最后确认从所有受支持 Beta 版本到 v1.0 的迁移，以及旧版本对较新 Schema 的安全拒绝。

## 退出条件

无 P0/P1，功能/Schema/API 冻结，全部 v1.0 发布清单和支持准备通过。
