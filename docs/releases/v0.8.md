# Octopus v0.8 版本执行文档

## 状态

已规划。

## 目标用户

在真实、长期、异构 Windows 环境中使用 Octopus 的公开 Beta 用户。

## 核心问题

安装、升级、规模、同步盘、权限、长路径和真实文件分布是否会破坏核心承诺。

## 用户价值

用户可将 Octopus 用于真实资料仓库，并在更新或环境异常时得到可预期的恢复路径。

## 范围

- 签名安装、自动升级、迁移 dry-run/备份/回滚和诊断包。
- 1k/10k/100k 规模硬化，同步盘/长路径/权限场景和崩溃数据收集（选择加入）。
- 公开反馈流程、已知问题和支持响应分类。

## 非目标

- v1.0 长期兼容性承诺、所有平台正式支持、不经审核的公开 Plugin 市场。

## 交付物

- 公开 Beta 安装包、升级/回滚系统、兼容矩阵、已知问题和 Beta 指标报告。

## 指标

- 更新成功率≥99%，未恢复的致命崩溃为 0，Raw 修改事件为 0。
- 发布门槛基准无未审批的>10%退化。

## 验收清单

- [ ] 前两个受支持 Beta 版本可原地升级并可回滚。
- [ ] 迁移失败不损坏上一个可用 Index 状态。
- [ ] 所有诊断数据默认本地，上传前脱敏并获得明确同意。

## 依赖

v0.4–v0.7 的安装、搜索、UI 和扩展能力均达到 Beta 质量。

## 风险

真实环境差异可能暴露长路径和同步盘边界；通过分阶段放量、远程禁用开关与可回滚发布缓解。

## 迁移要求

所有持久化变更必须有版本检测、dry-run、备份、回滚和旧版本拒绝误写较新 Schema 的能力。

## 退出条件

指标在代表性公开 Beta 样本中连续达标，升级/回滚和兼容矩阵验收完成。
