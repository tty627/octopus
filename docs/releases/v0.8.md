# Octopus v0.8 版本执行文档

## 状态

工程里程碑已完成。候选版本为 `0.8.0b1`；未进行公开 Beta 真人样本、真实历史安装器原地升级或签名产物验收，因此不声称已完成公开 Beta 发布。

## 目标用户

在长期、异构 Windows 环境中使用 Octopus，并需要可预期诊断和恢复路径的 Beta 用户。

## 核心问题

升级、迁移、规模、同步盘暂时不可读、权限和长路径是否会破坏 Raw 只读、节点身份和上一个可用 Index。

## 用户价值

用户可以在迁移或环境异常时获得校验备份、显式回滚、本地脱敏诊断和不会误删暂不可读子树的恢复路径。

## 范围

- 迁移 dry-run、SHA-256 备份、失败自动恢复和成功后显式回滚。
- 默认仅本地、无路径/内容/凭据的诊断包；手工分享前单独记录明确同意。
- v0.6/v0.7 来源的持久化契约矩阵、长中文路径、权限和同步盘暂不可读回归。
- 事务/增量/检索绝对门槛与相对 v0.7 的 10% 回归门禁。

## 非目标

- v1.0 长期兼容承诺、所有平台正式支持、不经审核的 Plugin 市场。
- 伪造签名证书、公开 Beta 人数、真实机器矩阵或历史安装器练习结果。

## 交付物

- `0.8.0b1` wheel/sdist/Windows 工程候选与[兼容矩阵](../product/COMPATIBILITY_MATRIX.md)。
- [`diagnostics create/inspect/prepare-share`](../user/DIAGNOSTICS_AND_RECOVERY.md) CLI、Local API 与桌面入口。
- `migrate --rollback RUN_ID` 及校验/防覆盖测试。
- 版本化性能基线和显式审批型回归比较器。

## 指标

- Raw 修改事件：0；越权/权限/长路径/迁移回滚相关自动化全部通过。
- 全仓测试 140 项通过（34+40+26+40）；Ruff 与 strict mypy（40 个源模块）通过。
- 单文件增量更新 P50/P95：36.43/37.99 ms，绝对门槛 30 s 通过，较 v0.7 P95 86.9 ms 无退化。
- 800 文件事务 P95：3.013 s，绝对门槛 5 s 通过；较记录值 2.769 s 退化 8.83%，低于 10%。
- 400→800 P50 比率：2.076，绝对门槛 2.5 通过；较历史 1.173 相对变化 76.96%，已显式批准。批准理由：400 档样本的固定开销/时序波动使比率分母不稳定，800 P95 与绝对增长门槛均通过；原始五次测量保存在本地工程报告。
- 10 任务检索：Top-5 100%、MRR 1.0、解释失败 0；60 任务既有基线保持 90%。

## 验收清单

- [x] v0.6/v0.7 的 global/repository/Markdown/API 持久化契约可被当前版本识别，且本版未改变持久化 Schema，具备 Schema 级回退条件。
- [x] 迁移失败恢复上一个状态；成功迁移可校验回滚，迁移后发生变化时拒绝覆盖。
- [x] 诊断数据默认本地且不含路径、文件名、仓库/节点 ID、查询、内容、消息或凭据；分享副本要求明确同意且不自动上传。
- [x] 超过 260 字符的中文路径可索引/搜索，Raw SHA-256 不变。
- [x] 暂不可读的同步子树保留身份和旧索引，不误记为删除；单文件权限失败保留旧 Leaf。
- [x] 性能比较器同时执行绝对阈值、10% 相对阈值和列名审批。
- [ ] 真实 v0.6/v0.7 签名安装器原地升级/回退、Windows 机器矩阵和公开 Beta 样本未执行；按用户决定不作为继续工程迭代的阻断项，也不声称通过。

## 依赖

v0.4–v0.7 的安装、搜索、桌面、Local API 和 Plugin 能力。

## 风险

同步盘实现、Windows 构建、杀毒软件和企业权限策略仍可能暴露自动化夹具未覆盖的差异。诊断包减少排障泄漏面，但不能替代真实环境验证；签名和历史安装器练习仍是正式外部发布前的人工/基础设施事项。

## 迁移要求

所有持久化变更必须先检测版本、默认 dry-run、创建校验备份并支持全目标回滚。未知较新 Schema 在写前拒绝；回滚不得覆盖迁移后已变化的文件，不得读写 Raw。

## 退出条件

升级来源契约、迁移回滚、诊断脱敏、长路径/权限/同步盘保护和性能/检索门禁均完成可复现工程验证。真实公开 Beta、签名和历史安装器练习被明确记录为未执行，而不是虚构为已通过。
