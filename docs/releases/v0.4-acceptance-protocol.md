# v0.4 Windows 验收协议

## 工件与环境

- `0.4.0a1`：5 名目标用户可用性试验，签名完整离线安装器。
- `0.4.0rc1`：与开发者独立的 20 名目标用户正式验收。
- Windows 11 Home/Pro x64 干净虚拟机，标准用户，无 Python、Node.js 和 API Key。
- 每轮先验证 SHA256、主程序/CLI/卸载器/安装器 Authenticode SHA-256 与 RFC 3161 时间戳。

## 首次任务

从启动安装器开始计时。受试者独立安装，使用示例或允许的测试资料，完成预检、索引和
搜索，并打开一条有效索引结果或原文件。观察者不代操作，只记录阶段时间、结果、稳定
错误码和计数。

每个候选版本还需执行：

1. 索引中安全取消并一键重试；确认 Raw 哈希/文件系统修改事件为零。
2. 模拟空间不足、权限不足、仓库锁定、解析失败和离线升级检查。
3. 运行卸载与静默卸载，确认后台进程停止且配置、Raw、Index、示例保留。
4. 重装并确认原仓库自动出现；执行 v0.3 配置与 Index 升级烟雾测试。
5. 执行 Defender 扫描并记录签名、版本资源和误报结果。

## 通过标准

- 正式验收至少 16/20 在 10 分钟内打开有效结果。
- 至少 19/20 首次任务成功。
- 20/20 Raw 修改事件为 0。
- 安装器静默安装/卸载、升级/重装和数据保留全部通过，无 P0/P1。

## 隐私与导出

本地记录只包含匿名 session ID、产品版本、是否示例、阶段时间、结果、错误码和文件计数；
不记录路径、文件名、查询、内容或 API Key。记录保存在
`%APPDATA%\Octopus\onboarding-runs`，只由受试者主动检查并导出，不进行遥测上传。
