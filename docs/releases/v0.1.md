# Octopus v0.1 版本执行文档

## 状态

已完成的内部里程碑；无正式 Git Tag，不追溯伪造发布日期。

## 目标用户

需要验证本地只读索引架构的项目内部用户和技术合作者。

## 核心问题

能否在不改写原始目录的情况下，为文件和文件夹建立可搜索、可点击的轻量索引层。

## 用户价值

证明用户可以保留原有文件组织，同时获得人和 Agent 共用的索引与链接。

## 范围

- Python 3.12+ Windows-first CLI。
- Raw/Index Repository 分离和 Raw 只读边界。
- Leaf、FolderNode、Manifest、SQLite FTS5 可重建缓存。
- PDF、Office、图片/OCR 解析，DeepSeek 抽象，本地搜索和 Markmap。
- 轮询 Watcher、稳定性判定、状态队列和用户区域保护。

## 非目标

- GUI、安装器、原生文件系统事件、向量数据库和完整长路径方案。

## 交付物

- 可运行 CLI 与基础测试集。
- [ADR 0001](../adr/0001-mvp-architecture.md) 与 v1.0.1 产品/文件格式规范。

## 指标

- 架构可行性为主；当时未建立可量化产品指标。
- Raw 快照测试的修改事件为 0。

## 验收清单

- [x] Raw/Index 分离可运行。
- [x] 可生成 Leaf 和自底向上 FolderNode。
- [x] 本地搜索库可删除重建。
- [x] 非文本原件只在建立/更新 Leaf 时读取。

## 依赖

Python、SQLite FTS5；可选 DeepSeek API 和 Markmap CLI。

## 风险

范围较大、首次安装面向开发者、缺少事务恢复与发布性能基线；分别在 v0.2–v0.4 中处理。

## 迁移要求

无；该里程碑未形成对外发布承诺。

## 退出条件

架构可行性已由端到端生成、搜索和 Raw 只读测试证明，已退出。
