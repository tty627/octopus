# Octopus v1.1 核心前端闭环

## 状态

`1.1.0.dev0` 开发实施版本；目标是 unsigned Windows 本机安装验证，不是正式签名发布。

## 目标用户

Windows 11 x64 上拥有大量异构本地资料、依赖既有文件夹工作并需要重新组合资料的个人知识工作者。

## 核心问题

旧桌面端以仓库和维护操作为中心，搜索结果无法自然承接证据核验、任务组合和受控导出。

## 用户价值

用户无需迁移或修改原文件，即可从任务出发找到证据、确认资料并保存为可执行任务包。

## 范围

WebView2 桌面壳层、资料空间向导、工作台、本地优先搜索、证据检查器、任务包、Markdown/Package 导出和基础健康恢复。

## 非目标

最近变化时间线、完整新鲜度差异、关系图、Agent 交接、跨资料空间搜索和独立 Markmap HTML。

## 交付物

React/TypeScript SPA、Python WebView2 壳层、Local API v1 增量端点、任务包持久化、自动化测试和 unsigned Inno 安装包。

## 指标

核心 E2E 流程通过；1440×900、1100×720、980×720 无空白、遮挡或横向溢出；Python 与前端门禁通过。

## 验收清单

- 本地结果先到，AI 失败不覆盖本地结果。
- 单击只检查证据，明确操作才加入任务包。
- revision 冲突、断线草稿、归档和导出行为可回归。
- 安装、启动、CLI/GUI 版本、仓库重发现和卸载数据保留可验证。

## 依赖

Windows 11 x64、系统 WebView2 Runtime、Local API v1、Python 索引引擎和 Package Plugin。

## 风险

开发安装包未签名；WebView2 缺失需要用户安装；正式 Defender 与干净机验证不在本轮门槛。

## 迁移要求

Local API contract 保持 `1.0`；Raw、Markdown Index 和现有仓库不迁移；可重建搜索缓存自动升级到 `0.6`。

## 退出条件

全仓测试、前端构建/E2E、PyInstaller smoke、unsigned Inno 构建和本机安装保留验证全部通过。
