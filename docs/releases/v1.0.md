# Octopus v1.0 版本执行文档

## 状态

已规划；General Availability 正式版。

## 目标用户

需要长期、稳定、可升级地管理本地混合文件资料的 Windows 个人用户、知识工作者和本地 Agent 工具。

## 核心问题

Octopus 是否已具备用户可以放心安装、长期更新、升级和获得支持的正式产品质量。

## 用户价值

用户在不改变原文件组织、不被迫依赖网络 AI 的情况下，持续获得可信、可解释、可点击的资料定位与组合能力。

## 范围

- 稳定的本地索引/检索核心、Windows 安装与桌面工作流。
- 可恢复更新、版本化迁移、运行/费用可观测和无 AI 降级。
- 已记录的支持格式、性能基线、兼容政策、Plugin 权限边界和支持流程。

## 非目标

- 云端多用户同步、所有操作系统正式支持、通用知识库替代品和任意未审核 Plugin 自动执行。

## 交付物

- 签名正式安装包、Tag/校验和、发布说明、用户/管理/恢复手册、兼容矩阵和公开支持政策。

## 指标

- Raw 修改事件为 0；核心更新成功率≥99%；故障注入恢复/回滚率 100%。
- 标准任务 Top-5 文件命中率≥80%，所有性能回归在已审批阈值内。
- 无 API Key 时安装、索引、更新、校验和本地搜索全部可用。

## 验收清单

- [ ] v0.9 RC 的全部清单在正式产物上重复通过。
- [ ] 安装、升级、回滚、备份、恢复和卸载文档在干净环境验证。
- [ ] 版本、Tag、工件、CHANGELOG、规范/Schema 说明和支持承诺一致。

## 依赖

v0.9 候选版无阻断问题，发布工程、签名、文档和支持能力均已就绪。

## 风险

1.0 后的兼容承诺会放大早期契约缺陷；只在 RC 期完成真实升级验证、契约审查和支持演练后发布。

## 迁移要求

支持从公开 Beta/RC 原地升级到 v1.0；迁移只写 Index/全局配置，提供 dry-run、备份和回滚，不要求删除并重建用户仓库。

## 退出条件

正式产物的所有发布、安全、性能、检索、升级和支持门槛通过，完成 `v1.0.0` Tag 与发布工件归档。
