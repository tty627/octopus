# Octopus v1.0 版本执行文档

## 状态

`1.0.0` 工程最终版本已完成。源码、冻结契约、测试和 unsigned 工程工件达到最终目标；未创建/推送正式 Tag，未完成 Authenticode、干净机/Defender、真实历史安装器或真人发布职责，因此不声称已完成对外 GA 发布。

## 目标用户

需要长期、稳定、可升级地管理本地混合文件资料的 Windows 11 x64 个人用户、知识工作者和本地 Agent 工具。

## 核心问题

Octopus 是否已具备用户可以稳定安装、持续更新、恢复、搜索、扩展和诊断的完整 1.0 工程质量，并对未验证外部门槛保持诚实边界。

## 用户价值

用户在不改变原文件组织、不依赖网络 AI 的情况下，持续获得可信、可解释、可点击的资料定位与 Package/Timeline 组合能力；故障时保留上一个 Index 状态并有明确恢复路径。

## 范围

- 本地索引/检索核心、Windows 桌面工作流和 Local API v1。
- 可恢复事务、版本化迁移/回滚、本地脱敏诊断和无 AI 降级。
- 支持格式、性能/检索基线、v1 兼容政策、Plugin API v1 权限边界和支持/回滚流程。

## 非目标

- 云端多用户同步、所有操作系统正式支持、通用知识库替代品、公开 Plugin 市场和任意未审核代码自动执行。
- 在没有证书、真实机器或真人参与时伪造签名/发布/验收结果。

## 交付物

- 版本为 `1.0.0` 的源码、wheel/sdist、Windows onedir 工程候选、Build Manifest 和 SHA256SUMS。
- [1.0 用户指南](../user/USER_GUIDE.md)、安装/排障、诊断/迁移恢复、兼容矩阵、支持政策和紧急回滚手册。
- v1 契约冻结、机器可读 Known Issues、自动发布审计和完整测试/性能/检索证据。

## 指标

- Raw 修改事件：0；故障注入恢复/回滚自动化：100%。
- 核心全仓自动化、Ruff、strict mypy、构建、GUI 烟雾和 release-audit 全部通过。
- 10 任务 Top-5 100%、MRR 1.0、解释失败 0；60 任务 Hit@5 90%，超过 80% 门槛。
- 无 API Key 时创建、索引、更新、校验、本地搜索、诊断和缓存修复可用。
- v0.9 性能复测：增量 P95 88.57 ms；800 文件事务 P95 4.613 s；全部绝对门槛通过，列名相对退化审批保留在性能基线。

## 依赖

v0.9 工程 RC 无开放 P0/P1，持久化/API/Plugin/诊断契约已冻结；外部 GA 发布仍依赖签名证书、Windows 验证环境和真人 Release Operator。

## 风险

未执行外部门槛意味着当前 `1.0.0` 是可复现的工程最终版本，而不是已签名分发的公开 GA。对外发布前仍必须从最终 commit 重建、签名、校验、安装/升级/回退/卸载、Defender 扫描并由真人确认支持职责。

## 迁移要求

支持从 v0.6–v0.9 工程 Beta/RC 契约原地进入 1.0；本次不改变 global/repository/Markdown Schema。后续 1.x 保持既有持久化字段语义，新增字段必须可选；破坏性变更需要新的主版本和显式备份/迁移/回滚。

## 退出条件

`1.0.0` 源码、冻结契约、自动化、检索/性能、文档和 unsigned 工程工件全部完成并可复现。正式 Tag、签名安装器和真人/真实环境发布检查保留为对外 GA 的明确外部门槛。
