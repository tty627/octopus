# Octopus v0.5 版本执行文档

## 状态

已规划。

## 目标用户

已能建立索引，但需要用 Octopus 稳定完成资料定位和阅读组合任务的知识工作者。

## 核心问题

搜索结果是否真正帮用户更快找到正确文件，并理解推荐、引用和阅读顺序。

## 用户价值

用户找到目标文件的时间相比文件管理器基线减少至少 50%。

## 范围

- 标准任务评测集、相关性标注和 Top-5 评估工具。
- 推荐原因、证据定位、状态/质量风险和一键打开。
- 增量搜索缓存刷新与 AI 降级/成本策略。

## 非目标

- 取代原生文件管理器、生成完整文档知识库、开放 Plugin SDK。

## 交付物

- 检索评测集和报告、可解释结果契约、增量搜索更新与用户研究报告。

## 指标

- 标准任务 Top-5 文件命中率≥80%。
- 目标文件定位中位时间减少≥50%；单文件增量更新 P95≤30秒。

## 验收清单

- [ ] 评测集覆盖中英文、文本/非文本、同名和过期索引。
- [ ] 每个 AI 结论只能引用给定候选，无有效证据时显式降级。
- [ ] 不配置 API Key 时基础搜索和打开链路仍可用。

## 依赖

v0.4 可用安装/首次流程，v0.3 证据定位和搜索可解释基础。

## 风险

评测集过小或过于理想化会虚高命中率；使用真实任务类型、盲标和版本化数据集缓解。

## 迁移要求

搜索排名或库 Schema 变更只重建派生缓存，不重写 Raw；评测结果必须携带算法/数据集版本。

## 退出条件

命中率、定位时间、增量延迟和无 AI 降级门槛全部达标。
