# Octopus v0.5 版本执行文档

## 状态

工程实现中。可解释搜索、增量缓存、60 任务离线评测和本地研究记录器已进入功能分支；
产品仍为 `0.4.0.dev0`，等待 v0.4 外部验收后才能建立 v0.5 集成分支。

## 目标用户

已能建立索引，但需要用 Octopus 稳定完成资料定位和阅读组合任务的知识工作者。

## 核心问题

搜索结果是否真正帮用户更快找到正确文件，并理解推荐、引用和阅读顺序。

## 用户价值

用户找到目标文件的时间相比文件管理器基线减少至少 50%。

## 范围

- 标准任务评测集、相关性标注和 Top-5 评估工具。
- 推荐原因、证据定位、状态/质量风险和一键打开。
- 增量搜索缓存刷新与 AI 降级/成本策略。

## 非目标

- 取代原生文件管理器、生成完整文档知识库、开放 Plugin SDK。

## 交付物

- 检索评测集和报告、可解释结果契约、增量搜索更新与
  [用户研究协议](v0.5-user-study-protocol.md)。真实用户研究报告仍是发布阻断项。

## 指标

- 标准任务 Top-5 文件命中率≥80%。
- 目标文件定位中位时间减少≥50%；单文件增量更新 P95≤30秒。

## 验收清单

- [x] 60 任务版本化评测集覆盖中英文、文本/非文本、同名、过期和质量风险。
- [x] 本地确定性基线 Top-5 命中率达到 91.7%（55/60；Windows 11、Python 3.13、
  60 任务、算法 `octopus-0.5-local-v1`）。
- [x] 每个 AI 结论只能引用给定候选及已有证据，无有效证据时显式本地降级。
- [x] 不配置 API Key 时 GUI、CLI 和 API 的基础搜索及打开链路仍可用。
- [x] 增量缓存刷新、Manifest generation 校验和失败后自动重建已有自动化测试。
- [x] 匿名本地研究记录器、汇总器和 20 人反平衡研究协议已实现。
- [ ] Windows 11/Python 3.12 参考环境单文件增量更新 P95≤30秒且无>10%关键回归。
- [ ] 至少 20 人受控研究达到目标文件定位中位时间减少≥50%。

## 依赖

v0.4 可用安装/首次流程，v0.3 证据定位和搜索可解释基础。

## 风险

合成集可能虚高命中率；以盲标、同名/过期/质量风险任务和独立 20 人受控研究缓解，正式
发布不得用离线命中率代替定位时间结果。

## 迁移要求

搜索排名或库 Schema 变更只重建派生缓存，不重写 Raw；评测结果必须携带算法/数据集版本。

## 退出条件

工程清单、参考环境增量门槛和真实研究全部通过；v0.4 已发布并完成 v0.5 分支/版本切换；
命中率、定位时间、增量延迟和无 AI 降级门槛全部达标。
