# Octopus v0.5 版本执行文档

## 状态

工程里程碑已完成；产品版本为 `0.5.0a1`。真人定位时间研究由产品负责人取消，不生成或
伪造用户数据；版本化离线任务、确定性检查步数代理、契约测试和增量基准均已通过。

## 目标用户

已能建立索引，但需要用 Octopus 稳定完成资料定位和阅读组合任务的知识工作者。

## 核心问题

搜索结果是否真正帮用户更快找到正确文件，并理解推荐、引用和阅读顺序。

## 用户价值

在离线任务中减少文件名基线的目标文件检查步数，并为每个结果提供可核验解释和打开目标。

## 范围

- 标准任务评测集、相关性标注和 Top-5 评估工具。
- 推荐原因、证据定位、状态/质量风险和一键打开。
- 搜索 Schema 自动重建与 AI 降级/成本策略。

## 非目标

- 取代原生文件管理器、生成完整文档知识库、开放 Plugin SDK。

## 交付物

- 检索评测集和工程报告、可解释结果契约、搜索缓存迁移和一键打开链路。

## 指标

- 标准任务 Top-5 文件命中率≥80%。
- 文件名基线检查步数减少≥30%；单文件增量更新 P95≤30秒。

## 验收清单

- [x] 评测集覆盖中英文、文本/非文本、同名和过期索引。
- [x] Top-5 100%、MRR 1.00、文件名基线检查步数减少 84.1%，10 个任务无失败。
- [x] 每个 AI 结论只能引用给定候选，无有效证据时显式降级。
- [x] API/CLI 返回源相对路径、打开 URI、字段命中、解析证据及风险标志。
- [x] 不配置 API Key 时基础搜索、评测和打开链路仍可用。
- [x] 单文件增量更新 P95 48.8 ms，低于 30 秒门槛。

## 依赖

v0.4 可用安装/首次流程，v0.3 证据定位和搜索可解释基础。

## 风险

评测集较小且由项目维护者标注，可能虚高命中率；报告明确限定为工程回归基线，未来增加
外部数据时必须递增数据集版本且不得覆盖历史结果。

## 迁移要求

搜索排名或库 Schema 变更只重建派生缓存，不重写 Raw；评测结果必须携带算法/数据集版本。

## 退出条件

工程门槛已达标，详见 [检索评测基线](../product/SEARCH_EVALUATION_BASELINE.md)。真人定位
时间未评测，也不作为后续版本开发的阻塞项。
