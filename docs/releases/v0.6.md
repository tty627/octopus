# Octopus v0.6 版本执行文档

## 状态

工程里程碑已完成；产品版本为 `0.6.0a1`。真人工作流研究按产品负责人决定不执行，退出
条件由 Local API 契约测试、桌面控制器工作流、Raw 不变性和无窗口 smoke test验证。

## 目标用户

需要可视化管理仓库、更新状态和搜索结果，且不希望直接使用 CLI 的普通用户。

## 核心问题

用户能否在桌面界面中完成核心工作流，并理解 pending、failed、orphaned、AI 费用和恢复状态。

## 用户价值

用户无需记忆命令即可建立仓库、搜索、打开结果、更新、校验和恢复。

## 范围

- 桌面端仓库列表、初始化、异步索引任务、搜索结果树和一键打开。
- 状态中心、一键重试/校验/修复、AI 用量和 Raw 只读信任提示。
- 桌面端仅通过已稳定的本地服务 API/CLI 边界调用核心，不复制索引逻辑。

## 非目标

- 远程多用户 SaaS、移动端、协作编辑和第三方 Plugin 市场。

## 交付物

- 桌面 Beta、[稳定 Local API v1 契约](../api/LOCAL_API_V1.md)、可访问性/错误处理测试和
  工程工作流报告。

## 指标

- 核心工作流 100% 可在 UI 中完成，不需 CLI。
- 桌面崩溃不损坏正在提交的 Index 事务；Raw 修改事件为 0。

## 验收清单

- [x] 桌面控制器仅调用 Local API，不导入 Engine、SearchIndex 或仓库配置写逻辑。
- [x] 仓库创建、列表、搜索、更新、失败重试、校验、搜索修复和打开均有 UI 路径。
- [x] UI 与 CLI 共用核心 API/Engine，状态、SearchReport、ValidationReport 和 Job 契约一致。
- [x] 服务未启动自动连接/启动；仓库锁、AI 降级和迁移需求均有可执行提示。
- [x] `Ctrl+F`、`Ctrl+N`、`F5` 键盘路径和 96/192 DPI 缩放契约通过测试。
- [x] 中文/空格 Windows 路径的 API 首建、异步任务及 Raw 字节不变通过自动化测试。
- [x] `octopus-gui --smoke-test` 无窗口启动验证通过。
- [x] 119 项测试、全仓 Ruff、Windows/Linux strict mypy 和 sdist/wheel 构建通过。
- [x] 单文件增量更新 P50/P95 为 85.6/86.9 ms，低于 30 秒门槛。

## 依赖

v0.5 稳定检索价值、v0.4 安装链路和稳定本地服务契约。

## 风险

UI 与 CLI 双重业务逻辑会产生状态偏差；桌面端只保留展示与工作流编排，所有仓库动作通过
`DesktopController`/Local API v1 完成，并以契约版本握手拒绝不兼容服务。

## 迁移要求

复用现有 Index Repository 和全局配置；UI 首次启动执行只读兼容检查，需迁移时先展示计划。

## 退出条件

工程退出条件已通过。未执行真人工作流研究、交互式视觉走查、干净虚拟机或签名工件验证，
也不声称相应真实用户/发布指标；这些项目不阻塞后续工程版本迭代。
