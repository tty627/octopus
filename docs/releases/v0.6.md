# Octopus v0.6 版本执行文档

## 状态

已规划。

## 目标用户

需要可视化管理仓库、更新状态和搜索结果，且不希望直接使用 CLI 的普通用户。

## 核心问题

用户能否在桌面界面中完成核心工作流，并理解 pending、failed、orphaned、AI 费用和恢复状态。

## 用户价值

用户无需记忆命令即可建立仓库、搜索、打开结果、更新、校验和恢复。

## 范围

- 桌面端仓库列表、初始化、索引进度、搜索、结果地图和一键打开。
- 状态中心、一键重试/校验/修复、AI 用量和 Raw 只读信任提示。
- 桌面端仅通过已稳定的本地服务 API/CLI 边界调用核心，不复制索引逻辑。

## 非目标

- 远程多用户 SaaS、移动端、协作编辑和第三方 Plugin 市场。

## 交付物

- 桌面 Beta、稳定本地 API 契约、UI 可访问性/错误处理测试和工作流研究报告。

## 指标

- 核心工作流 100% 可在 UI 中完成，不需 CLI。
- 桌面崩溃不损坏正在提交的 Index 事务；Raw 修改事件为 0。

## 验收清单

- [ ] UI 与 CLI 对同一仓库产生等价状态和报告。
- [ ] 服务未启动、仓库被锁、AI 失败和迁移需求在 UI 中有可执行提示。
- [ ] 键盘导航、高 DPI 和常见 Windows 路径场景通过。

## 依赖

v0.5 稳定检索价值、v0.4 安装链路和稳定本地服务契约。

## 风险

UI 与 CLI 双重业务逻辑会产生状态偏差；通过单一核心服务和契约测试禁止重复实现。

## 迁移要求

复用现有 Index Repository 和全局配置；UI 首次启动执行只读兼容检查，需迁移时先展示计划。

## 退出条件

核心工作流、异常状态、可访问性和 UI/CLI 等价性验收通过。
