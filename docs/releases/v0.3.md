# Octopus v0.3 版本执行文档

## 状态

已完成；正式版本为 `0.3.0`。

## 目标用户

在中大型混合文件仓库上试用 Octopus，并需要检索结果具有解析证据和可解释排序的早期用户。

## 核心问题

当仓库和 FolderNode 规模增长时，索引更新是否仍然稳定，搜索摘要是否可追溯到真实的页、章节、Sheet、Slide 或 OCR 证据。

## 用户价值

用户可以在更大资料集上获得不因 SQLite/事务规模而失败的检索，并能理解结果来自哪个索引证据。

## 范围

- 去重和分批扩展 FolderNode 子节点，消除 SQLite 参数上限失败。
- 减少事务记录写放大，建立 1k/10k/100k 标准数据集和基准流程。
- 解析证据定位符、提取统计、截断状态和有界输入。
- 搜索字段权重、精确名称增益、查询词覆盖与候选验证引用。
- Prompt 版本、调用/Token/成本限制和搜索缓存迁移。

## 非目标

- 面向普通用户的安装器和初始化向导。
- 正式桌面 UI、已稳定公开服务 API 或 Plugin SDK。
- 对所有文件格式、所有硬件和 100k 文件 AI 全量处理作出性能承诺。

## 交付物

- 可复现的性能基线和规模回归用例。
- 证据化解析、可解释搜索、SQLite 分批与事务优化。
- [ADR 0003](../adr/0003-grounded-search-and-parser-evidence.md) 及当前版本报告。

## 指标

- 50,000 个子节点扩展无 SQLite 参数失败，单批≤1,000。
- 参考环境 800 文件事务总耗时≤5秒，400→800 增长比≤2.5。
- Raw 修改事件为 0；故障注入恢复/回滚率 100%。

## 验收清单

- [x] 50,000 子节点回归用例通过。
- [x] 事务完成 1 次预热和 5 次正式测量，耗时和增长比门槛通过。
- [x] 解析证据、搜索引用、Prompt/Token/成本边界用例通过。
- [x] pytest、覆盖率、Ruff、strict Mypy、sdist/wheel 构建全部通过。
- [x] 版本源、CLI、包元数据、RunReport 和生成器版本一致。

## 依赖

v0.2 事务边界、可重建搜索缓存和 v1.0.1 Leaf/FolderNode 规范。

## 风险

- 事务优化不得减少目标变更前的回滚意图持久化。
- 检索证据质量受解析器能力限制；用质量标志和截断说明降级，不伪造证据。
- 当前工作区已存在面向后续 UI 的 API/服务原型；本版本不对其外部稳定性作承诺。

## 迁移要求

搜索缓存是派生数据，在 Schema 不匹配时自动重建。已有 generator/parser version 保留；新生成内容使用当前产品版本。

## 退出条件

所有验收项通过，性能基线记录可重复结果，无已知 P0/P1 正确性或数据安全问题。
