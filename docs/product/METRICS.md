# Octopus 产品指标规范

## 指标原则

- 指标只记录完成产品决策所需的最小数据，默认不上传文件内容、路径、摘要、API Key 或搜索原文。
- 本地运行报告是工程指标的首要数据源；用户研究指标使用匿名任务结果。
- 版本发布文档必须声明适用的指标、样本数、环境和结果。

## 核心指标

| 类别 | 指标 | 定义 | 数据源 | 门槛 |
| --- | --- | --- | --- | --- |
| 安全 | Raw 修改事件 | Octopus 运行前后 Raw 文件 hash/结构变化 | E2E 快照 | 始终为 0 |
| 激活 | 首次有效结果时间 | 开始安装到用户打开首个有用索引链接 | 受控用户测试 | v0.4 起80% 用户≤10分钟 |
| 可靠性 | 更新成功率 | 非用户取消的 success/partial 运行占比 | RunReport | ≥99% |
| 恢复 | 中断恢复率 | 故障注入后自动恢复或回滚通过数/用例数 | 自动化测试 | 100% |
| 检索 | Top-5 文件命中率 | 标准任务的目标文件出现在前 5 条的比率 | 离线评测集 | v0.5 起≥80% |
| 效率 | 定位时间改善 | 使用 Octopus 与文件管理器基线的任务中位时间差 | 受控用户测试 | v0.5 起减少≥50% |
| 性能 | 单文件增量更新 P95 | 稳态仓库修改一个支持文件后到索引可搜索 | 标准基准 | ≤30秒 |
| 性能 | 关键基准回归 | 与上一个通过版本的差异 | 标准基准 | 退化>10%需审批 |
| AI | 调用、Token、耗时、估算成本 | 每次运行的聚合使用量 | RunReport/SearchReport | 100%可观测 |
| 留存 | 周活跃仓库留存 | 首次成功后第 2/4 周仍发生有效搜索或更新 | 选择加入的匿名研究 | Beta 期建立基线 |

## 状态分类

- `success`：事务、Manifest 和派生索引完成，无节点失败。
- `partial`：可用结果已提交，但存在可解释的节点或派生缓存失败。
- `failed`：本次运行未产生新的已承诺状态。
- 用户主动取消、无变更 dry-run 和测试运行不纳入成功率分母。

## 发布报告

每个 Beta/RC/GA 发布候选必须生成一份指标摘要，包含样本量、环境、数据时间范围、结果和未达标项。不得只报告平均值；延迟类指标至少报告 P50/P95。

## v0.5 检索评测口径

- 离线门槛使用 `octopus-retrieval-v1` 的 60 个盲标任务和本地确定性模式；Hit@5 按目标
  `raw_relative_path` 是否进入前五计算，报告必须携带数据集、算法和产品版本。
- 反平衡受控研究工具保留，本地记录不得包含查询、路径、URI 或内容；v0.5 起真人研究由
  产品负责人取消，不采集、不推断，也不作为后续工程版本阻断项。
- AI 结果可作为次要分析，不得代替无 API Key 的本地命中率门槛。
