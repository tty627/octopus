# Octopus v0.8 兼容矩阵

本矩阵描述 `0.8.0b1` 的工程支持边界。`octopus compatibility` 可输出同一组机器可读信息。

## 平台

| 环境 | 状态 | 说明 |
| --- | --- | --- |
| Windows 11 x64、NTFS、本地每用户安装 | 支持 | 自动化、长路径和打包主环境 |
| OneDrive/同步盘中的 Raw | 有条件支持 | 暂时不可读的子树保留原节点并标记 `scan_access_denied`，不会误判删除 |
| 同步盘中的 Index | 不推荐 | 事务依赖同卷原子替换；同步工具仍可能增加锁冲突 |
| Windows 10、网络盘、FAT/exFAT | 未承诺 | 可运行不等于正式支持，v0.8 未做真实机器矩阵 |
| macOS/Linux | 开发可运行 | 不是 v1.0 正式支持平台 |

## 产品升级与二进制回退

| 来源 | 到 `0.8.0b1` | 回退到来源版本 | 持久化迁移 |
| --- | --- | --- | --- |
| `0.6.0a1` | 契约兼容 | Schema 角度安全 | 无 |
| `0.7.0a1` | 契约兼容 | Schema 角度安全 | 无 |

两条来源都使用 global/repository/Markdown Schema `0.2` 和 Local API v1，因此当前工程夹具不需要持久化迁移。此表证明的是文件契约兼容性，不等同于在两套历史签名安装器之间完成了真实原地安装练习。

## 契约

| 契约 | 当前版本 | 稳定性 | 回退策略 |
| --- | --- | --- | --- |
| 全局配置 | `0.2` | 稳定 | 迁移前校验备份；成功迁移可按 run ID 显式回滚 |
| 仓库配置/Manifest | `0.2` | 稳定 | 新版本 Schema 在写前拒绝；迁移不得写 Raw |
| Markdown Index | `0.2` | 稳定 | v1.0 起承诺同主版本向前兼容 |
| Local API | `1.0` | 稳定 | v1 内仅增加端点/可选字段 |
| Plugin API | `1.0` | 开发者预览 | 不兼容范围在启动 Worker 前禁用 |
| SQLite 搜索缓存 | `0.5` | 可重建 | 删除缓存后从已提交 Index 重建 |

## 长路径、权限与同步状态

- 自动化覆盖绝对路径超过 260 字符的中文目录、索引、搜索与 Raw SHA-256 不变。
- 已索引子树暂时无读取权限时，节点身份和已有 Index 保留为 `pending_edit/scan_access_denied`；权限恢复后可重试。
- 单文件解析遇到 `PermissionError` 时，本次报告为 `partial`，旧 Leaf 保留，Raw 字节不变。
- Office `~$` 锁文件、`.tmp/.lock` 临时文件和重解析点继续按稳定性/忽略策略处理。
